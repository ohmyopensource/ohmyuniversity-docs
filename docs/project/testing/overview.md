# Overview

## Testing Overview

The testing documentation describes how OhMyUniversity verifies the correctness, reliability and usability of the implemented system.

The backend platform is already largely documented and covered by its own test activities. For this reason, the testing focus of this document is placed mainly on the web and mobile clients, while still including the integration points with the backend services.

The main areas covered by the testing activity are:
- authentication and user access;
- student dashboard and academic data visualization;
- career data, averages and CFU progress;
- exam sessions and bookings;
- digital badge and profile data;
- orientation pages and future planning flows;
- external university services;
- calendar and timetable sections;
- interaction between frontend clients and backend APIs.

The goal is to ensure that the implemented features behave according to the requirements defined in the RAD and remain stable during future iterations.
