# OhMyUniversity! - RAD: 3.4 System Models

This section represents the technical formalization of the previously described requirements. The purpose of the system models is to bridge the gap between the discursive understanding of the requirements and the technical design, using the standard UML (Unified Modeling Language) to eliminate the ambiguities typical of natural language.

Following an object-oriented approach, the **OhMyUniversity!** system is analyzed through three complementary perspectives:

- **Functional Model (Sections 3.4.1 and 3.4.2):** Describes the system's behavior from the user's point of view, defining what the system does through scenarios and use cases.

- **Object Model (Section 3.4.3):** Describes the static structure of the system, identifying entities, classes, their attributes, and the relationships that connect them.

- **Dynamic Model (Section 3.4.4):** Describes the internal behavior of the system over time, illustrating how objects interact with each other to implement the functionalities described in the use cases.

The set of these models forms the basis for the subsequent System Design phase and ensures that every functionality is verifiable and testable against the established objectives.
